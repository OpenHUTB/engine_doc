本节将介绍 nDisplay启动 用户界面中所有的可用设置和选项。

启动器选项卡


点击查看大图。

功能按钮

说明

渲染API（Render API）

指定下次点击"运行（Run）"时使用的渲染API。

渲染模式（Render mode）

指定nDisplay在每个群集节点上生成的输出类型： 

  • 单视场- 以相机视点对场景进行单视场渲染。

  • 帧顺序（Frame sequential）- 有源四缓冲立体声。

  • 并排（Side-by-side）- 无源水平对齐立体声

  • 从上到下（Top-bottom）- 无源垂直对齐立体声

单视场无需任何特定硬件功能，但帧顺序需要。确保显示设备、GPU和驱动程序设置与选择的渲染模式兼容。

使用所有可用核心（Use All Available Cores）

强制各虚幻引擎实例使用其主机上所有可用的处理器。
选中该选项时，nDisplay启动器 将在用于启动所有实例的命令行中添加 USEALLAVAILABLECORES 选项。

无纹理流送（No Texture Streaming）

禁用各虚幻实例的纹理流送。固定加载最高质量纹理。
选中该选项时，nDisplay启动器 将在用于启动所有实例的命令行中添加 NOTEXTURESTREAMING 选项。

自定义命令行参数（Custom command line arguments）

如需 nDisplay启动器命令行参数

自定义ExecCmds（Custom ExecCmds）

如需 nDisplay启动器 传递在启动时要执行的虚幻引擎控制台命令，请在此输入此类命令。nDisplay启动将使用 -ExecCmds 命令行参数将此类命令传到虚幻引擎的所有实例。

应用程序（Applications）

列出可使用 nDisplay 启动器 运行的所有已打包虚幻应用程序。使用面板下的 添加 和 删除第3步.打包和部署

配置文件（Config Files）

列出已设置的 nDisplay启动器 所有配置文件。使用下拉框右侧的 添加 和 删除第3步.打包和部署

运行（Run）

尝试连接到文件中配置的各群集节点（在 配置文件 中选择），并告知 nDisplay Listener 应用程序以启动在 应用程序 列表中选择的已打包虚幻引擎应用程序。

终止（Kill）

尝试连接到文件中配置的各群集节点（在 配置文件 中选择），并告知 nDisplay Listener 应用程序以关闭在 应用程序 列表中选择的已打包虚幻引擎应用程序。

群集事件选项卡

可使用 群集事件（Cluster events） 选项卡设置新的群集事件并将它们发送到nDisplay群集。

另参阅从nDisplay启动器发射群集事件。


点击查看大图。

功能按钮

说明

新建

点击以在 群集事件编辑器 中设置新群集事件，并将其添加到列表视图中。

参阅从nDisplay启动器发射群集事件。

修改

如在列表视图内选中群集事件，点击 修改 可在 群集事件编辑器 中打开该群集事件，以编辑其值。

删除

如在列表视图内选中群集事件，点击 删除 可将该群集事件从列表中删除。

发送

如在列表视图内选中群集事件，且nDisplay群集当前正在运行，点击 发送 可将选中群集节点发送到nDisplay群集。检查输出日志面板以查看主节点（master node）发回的响应。

日志选项卡

nDisplay默认将各源中的日志消息保存到已打包应用程序旁的 Saved/Logs 文件夹。可使用 日志 选项卡来配置不同源中的信息级别，此信息由nDisplay记录。


点击查看大图。

功能按钮

说明

使用自定义日志设置（Use custom log settings）

选中时nDisplay将使用此页上设置的不同源的日志级别（左侧列出）。

适用于每项（For each）

点击此类按钮来设置左侧列出的所有源，记录所选严重性或更高严重性的消息。

日志输出面板

日志输出面板将列出 nDisplay启动器 中执行的所有命令结果。


点击查看大图。

功能按钮

说明

复制

将左侧日志窗口中列出的所有消息复制到剪贴板。

清理

清除左侧日志窗口中的所有消息。
